# 0001. 采用学习优先的分层架构

- Status: Accepted
- Date: 2026-07-15

## Context

项目既要跟随教程理解 Deep Research 的内部机制，也要逐步形成可部署 Demo。若一开始依赖大型 Agent 框架，ReAct 循环、状态更新、上下文控制和工具执行边界会被隐藏；若所有实现都写在章节脚本里，又难以演进成可靠系统。

## Decision

采用“章节 Demo + 可复用核心”的分层架构：

- 第 1–4 章显式实现 ReAct/IterResearch、工具协议、记忆和检查点；
- `demos/` 只承担教学入口，复用逻辑进入 `src/deep_research/`；
- 模型、工具、存储均通过接口和适配器隔离；
- LangGraph 等框架只在理解核心机制后作为可选适配或对照实验；
- 训练依赖独立分组，基础运行不依赖 GPU 栈；
- 先完成本地可验证实现，再增加 Web、数据库和部署组件。

## Consequences

正面影响：核心机制透明、单元测试简单、供应商可替换、章节演进可追踪。

代价：早期需要自行实现部分状态机、检查点和工具网关能力；后期接入框架时可能存在适配工作。

## Alternatives

- 一开始使用 LangChain/LangGraph：开发更快，但削弱学习目标并增加框架耦合；
- 全部使用独立脚本：最容易上手，但难以测试、复用和上线；
- 一开始建设完整微服务：生产形态更完整，但会显著拖慢核心 Agent 验证。
